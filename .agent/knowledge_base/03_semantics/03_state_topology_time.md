# State, Topology, and Time

## Purpose

Separate graph state, structural topology, temporal ordering, and visibility metadata.

## Canonicality

Canonical for state and topology discipline.

## Rules

- Topology describes structural relation.
- State describes contingent evaluated condition.
- Time describes ordering, version, or event progression.
- Visibility and projection metadata describe access or view, not meaning.
- Transition direction may be modeled numerically or compositionally,
  but the doctrine defines the morphism semantics while contingent
  coordinates stay in the value layer.
