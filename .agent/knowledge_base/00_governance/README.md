# Governance Layer

## Purpose

This layer governs the knowledge base itself.

It defines:

- what counts as canon
- how files are read in order
- how concepts are promoted
- how legacy content maps into the new structure

## Canonicality

This folder is canonical for process and authority rules.
It does not define domain ontology except where needed to govern ontology publication.

## Files

- `canonicality_contract.md` — role and authority boundaries
- `read_order.md` — normative reading order
- `promotion_contract.md` — promotion gates from source/candidate to canon
- `migration_map.md` — old source to new destination mapping
